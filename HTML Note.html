Create a new HTML file

  1. HTML tags give a webpage its structure.
    You can use HTML tags to add photos, buttons, and other elements to your webpage.
  2. All pages should begin with <!DOCTYPE html>.
    This special string is known as a declaration and ensures the browser tries to meet industry-wide specifications.
  3. Notice that the entire contents of the page are nested within an html element.
    All other elements must be descendants of this html element.
    You must to add the lang attribute with the value 'language' to the opening html tag to specify that the language of the page is 'language'.
  4. Add a head element, which mainly consists of the file's character encoding, keywords, scripts, and other backend data.
    The content inside the head tag will not be displayed on the web page.
  5. You can set browser behavior by adding self-closing meta elements in the head.
    Inside the head element, nest a meta element with an attribute named charset.
    Set to the value to utf-8 which tells the browser how to encode characters for the page.
  6. For the styling of the page to look similar on mobile as it does on a desktop or laptop, you need to add a meta element with a special content attribute within the head element.
    For example:<meta name="viewport" content="width=device-width, initial-scale=1.0">
  7. The title element：it determines what browsers display in the title bar or tab for the page.
    One of the several elements that provide additional information not visible on the webpage, but are useful for search engines or determining how the page is displayed.
    It can be displayed in two ways:
    1. In the title bar when the page is open.
    2. In the browser tab, the page title appears when you hover over it.
      Even if the tab is not active, the title text is displayed when you hover over the tab.
  8. All page content elements that should be rendered to the page go inside the body element.
    However, other important information goes inside the head element.

How to quickly establish the basic structure of html files

  • After entering the exclamation point, press the tab key to quickly create the basic structure of the html file.
  • You can modify the language in html lang; modify the file name at the title.

Style

You have been limited regarding the presentation and appearance of the content you create.
To start taking control, add a style element within the head element.
You can add style to an element by specifying it in the style element and setting a property.
  • To directly write CSS syntax within the HTML document, it needs to be placed within the style tag.
  • To write CSS syntax within the head of HTML, it needs to be enclosed within the style tag.

Script

Scripts are pieces of code written in programming languages like JavaScript that can add interactivity, animations, and dynamic behavior to web pages. They are typically included in the HTML document using the script element.
Scripts can be used for a wide variety of purposes, such as:
網頁中的Script是用程式語言(如JavaScript)編寫的代碼片段,可以為網頁添加互動性、動畫和動態行為。它們通常使用script元素包含在HTML文檔中。Script可以用於各種用途,例如:
  1. Responding to user events. (clicks, mouse movements, keyboard input, etc.) 響應用戶事件
  2. Manipulating and modifying the content and structure of the page dynamically. 動態操作和修改頁面內容和結構
  3. Validating form data before submission. 在提交之前驗證表單數據
  4. Creating animations and visual effects. 創建動畫和視覺效果
  5. Retrieving and updating data from servers. (AJAX) 從服務器檢索和更新數據
  6. Controlling multimedia content like videos and audio. 控制多媒體內容,如視頻和音頻
  7. Providing enhanced user experiences and interactions. 提供增強的用戶體驗和交互
Scripts can be either embedded directly within the HTML file or loaded from an external file. External scripts are preferred as they promote code reusability and separates concerns between structure (HTML), presentation (CSS), and behavior (JavaScript).
Script可以直接嵌入HTML文件中,也可以從外部文件加載。更好的做法是使用外部Script,因為它促進了代碼重用,並將結構(HTML)、表現(CSS)和行為(JavaScript)分開。

Comment

Commenting allows you to leave messages without affecting the browser display. It also allows you to make code inactive. A comment in HTML starts with <!--, contains any number of lines of text, and ends with -->. For example, the comment <!-- TODO: Remove h1 --> contains the text TODO: Remove h1.

HTML elements and attributes

HTML elements basics

  • HTML elements have opening tags like <h1> and closing tags like </h1>.
    The text for an element goes between its opening and closing tags.
  • HTML5 has some elements that identify different content areas.
    These elements make your HTML easier to read and help with Search Engine Optimization (SEO) and accessibility.
  • Nesting and indantation：Put the element inside the main element. This is called nesting.
    Nested elements should be placed two spaces further to the right of the element they are nested in.
    This spacing is called indentation and it is used to make HTML easier to read.

Typography and Layout

  1. Heading：The h1 through h6 heading elements are used to signify the importance of content below them.
    The lower the number, the higher the importance, so h2 elements have less importance than h1 elements.
    Only use one h1 element per page and place lower importance headings below higher importance headings.
    When you add a lower rank heading element to the page, it's implied that you're starting a new subsection.
  2. section：Before adding any new content, you should make use of a section element to separate the photos content from the future content.
  3. br：Line break
  4. hr：Divider
    hr elements are self closing.
  5. span：一個內聯元素（inline element），通常用於將文本的一部分樣式化，或者用於將行內元素包裹起來以應用 CSS 樣式。
    通常用於將文字或行內元素（如超連結、圖像等）包裹起來，以便對它們進行樣式設置。
  6. div：一個區塊級元素（block element），通常用於組織和佈局 HTML 內容，並且可以包含其他的 HTML 元素。
    通常用於創建獨立的區域，並且可以對這些區域進行樣式設置和操作。

Text on websites

  • The p element is used to create a paragraph of text on websites.
  • The article element commonly contains multiple elements that have related information.
  • The em element is used to italicize text, and the strong element is used to bold text when emphasizing fonts.

Order List

  1. <ul> is used to start an unordered list, and each item in the list is represented by <li> (list item) tags.
  2. <ol> is used to start an ordered list, and each item in the list is represented by <li> (list item) tags. The numbering or sequencing of the items is handled automatically by the browser.
  3. <li> is used to represent each item in a list. It should be nested inside either a <ul> or <ol> element to indicate whether it's part of an unordered or ordered list, respectively.

Form

  • Teh fieldset element：The fieldset element is used to group related form controls together, and is often combined with the legend element to provide a label or caption for the group.
  • The label element：The label element is used to provide a descriptive label for form controls, such as input fields, checkboxes, and radio buttons.
    It helps improve accessibility by associating the label with the corresponding form control using the `for` attribute.
  • The select element：The select element is used to create a dropdown list or a list box, allowing users to choose one or more options from a predefined list.
    It provides a compact way to present many options in a small space.
  • The option element：The option element represents each individual choice within a select element's dropdown list or list box.
    It specifies the value and label for each option that users can select.
    Multiple option elements can be nested inside a single select element to create the list of choices.
    • If the value attribute is not specified in the option element, when the user selects the option from the drop-down menu, the value sent to the server will be the text content in the option element. If the value attribute is specified, the specified value will be returned. This can distinguish between the text displayed to the user and the value actually sent to the server, making the program clearer and easier to maintain.

    • submit：The submit type is used to create a submit button. When the user clicks on this button, the form data is submitted to the server for processing.

  5. The minlength attribute：Certain types of attribute values come with built-in form validation. For example, the "type" attribute set to "email" requires the value to be a valid email address.
    Add custom validation to the password input element by including a "minlength" attribute with a value of 8.
    By doing so, it prevents the submission of inputs with fewer than 8 characters.
  6. The pattern attribute：
    7. When using type="password", you can utilize the pattern attribute to specify a regular expression that the password must adhere to in order to be deemed valid. This can replace the length limit set by the minlength attribute. Here's the English translation for the systematic way of setting up patterns in regular expressions:
    1. Anchors
      • ^ Matches the beginning of a string
      • $ Matches the end of a string
      • \b 可以取代 ^ 和 $作為模式的開始和結束。
        差別在於: ^ 和 $ 是行的開始和結束，\b 是單詞邊界；使用 ^ 和 $ 不要求模式前後有單詞字符以外的字符，使用 \b 要求模式前後至少有一個非單詞字符。
    2. Quantifiers
      • * Matches the preceding subpattern zero or more times
      • + Matches the preceding subpattern one or more times
      • ? Matches the preceding subpattern zero or one time
      • {n} Matches the preceding subpattern exactly n times
      • {min,} Matches the preceding subpattern at least min times
      • {min,max} Matches the preceding subpattern at least min times, but no more than max times
    3. Character Classes
      • [abc] Matches any character inside the brackets
      • [^abc] Matches any character not inside the brackets
      • [a-z] Matches any character in the specified range
      • . Matches any character except newline
    4. Grouping and Back References
      • (abc) Groups multiple patterns as a single unit
      • \1 Matches the same text as the first capturing group
    5. Other Special Characters
      • \d Matches any digit character
      • \D Matches any non-digit character
      • \w Matches any word character (alphanumeric & underscore)
      • \W Matches any non-word character
      • \s Matches whitespace characters
      • \S Matches non-whitespace characters
    These elements can be combined as needed to construct the desired pattern. For example, to match an email address pattern, you can use: \b[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}\b
    This pattern ensures the email format is name@domain.com or a similar format.
  7. The checked attribute：此為檢查是否有正確輸入，若使用此屬性則必須輸入正確才會送出。

    Build a form

    1. You can build a web form to collect information from users.
    2. In the form element, add an action attribute that indicates where form data should be sent.
    3. The input element allows you several ways to collect data from a web form.
      Like img elements, input elements are self-closing and do not need closing tags.
    4. There are many kinds of inputs you can create using the type attribute.
      Specifying the type attribute of a form element is crucial for the browser to understand what type of data it should anticipate.
      You can easily create a password field, reset button, or a control to let users select a file from their computer.
    5. In order for a form's data to be accessed by the location specified in the action attribute, you must give the text field a name attribute and assign it a value to represent the data being submitted.
      For example, you could use the following syntax for an email address text field: <input type="text" name="email">
    6. Placeholder text is used to give people a hint about what kind of information to enter into an input.
    7. To prevent a user from submitting your form when required information is missing, you need to add the required attribute to an input element.
      There's no need to set a value to the required attribute.
      Instead, just add the word required to the input element, making sure there is space between it and other attributes.
    8. Use the button element to create a clickable button. For example, <button>Click Here</button> creates a button with the text Click Here.
      The default behavior of clicking a form button without any attributes submits the form to the location specified in the form's action attribute.
    9. Even though you added your button below the text input, they appear next to each other on the page.
      That's because both input and button elements are inline elements, which don't appear on new lines.
    10. The button you added will submit the form by default. However, relying on default behavior may cause confusion.
      Add the type attribute with the value 'submit' to the button to make it clear that it is a submit button.
    11. You can use radio buttons for questions where you want only one answer out of multiple options.
    12. label elements are used to help associate the text for an input element with the input element itself (especially for assistive technologies like screen readers).
      For example, <label><input type="radio"> sample</label> makes it so clicking the word 'sample' also selects the corresponding radio button.
    13. The id attribute is used to identify specific HTML elements. Each id attribute's value must be unique from all other id values for the entire page.
      When elements have multiple attributes, the order of the attributes doesn't matter.
    14. Notice that both radio buttons can be selected at the same time.
      To make it so selecting one radio button automatically deselects the other, both buttons must have a name attribute with the same value.
    15. If you select one of the radio buttons and submit the form, the form data for the button is based on its name and value attributes.
      Since your radio buttons do not have a value attribute, the form data will include select1-select2 = on, which is not useful when you have multiple buttons.
      So you need to add a value attribute to both radio buttons.
    16. The fieldset element is used to group related inputs and labels together in a web form. fieldset elements are block-level elements, meaning that they appear on a new line.
    17. The legend element acts as a caption for the content in the fieldset element.
      It gives users context about what they should enter into that part of the form.
    18. Forms commonly use checkboxes for questions that may have more than one answer.
      For example, here's a checkbox with the option of 'text': <input type="checkbox"> 'text'.
    19. There's another way to associate an input element's text with the element itself.
      You can nest the text within a label element and add a 'for' attribute with the same value as the input element's id attribute.
    20. When using the input element, adding the name and id attributes to it, although you may not notice this in the browser, can make it easier for the server to handle your web form, especially when there are multiple checkboxes.
    21. Like radio buttons, form data for selected checkboxes are name / value attribute pairs.
      While the value attribute is optional, it's best practice to include it with any checkboxes or radio buttons on the page.
      For convenience, set each checkbox's value attribute to the same value as its id attribute.
    22. In order to make a checkbox checked or radio button selected by default, you need to add the checked attribute to it.
      There's no need to set a value to the checked attribute.
      Instead, just add the word checked to the input element, making sure there is space between it and other attributes.
      Make the first radio button and the first checkbox selected by default.

    For example

    
<section>
    <h2>Dog Form</h2>
    <form action="URL"> <!--Step 1~2-->
        <fieldset> <!--Step 16-->
            <legend>Is your dog an indoor or outdoor dog?</legend> <!--Step 17-->
            <label><input id="indoor" type="radio" name="indoor-outdoor" value="indoor" checked> Indoor</label> <!--Step 11~15、22-->
            <label><input id="outdoor" type="radio" name="indoor-outdoor" value="outdoor"> Outdoor</label> <!--Step 11~15-->
        </fieldset> 
        <fieldset> 
            <legend>What's your dog's personality?</legend> 
            <input id="loving" type="checkbox" name="personality" value="loving" checked> <label for="loving">Loving</label>  <!--Step 18~22-->
            <input id="lazy" type="checkbox" name="personality" value="lazy"> <label for="lazy">Lazy</label>
            <input id="energetic" type="checkbox" name="personality" value="energetic"> <label for="energetic"> Energetic</label>
            <input type="text" name="dogphotourl" placeholder="dog photo URL" required> <!--Step 3~7-->
            <button type="submit">Submit</button> <!--Step 8~10-->
        </fieldset>
    </form>
